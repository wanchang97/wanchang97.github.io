# 人生写作 SOP（Standard Operating Procedure）

> 本站地址：**[https://wanchang97.github.io](https://wanchang97.github.io)**
> 这是一个同时面向「自我」、「世界」与「未来可能性」的长期写作与思想实验项目。

---

## 一、这个网站存在的三重目的

1. **私人精神档案**：记录真实的感受、困惑、孤独、时间与命运的经验。
2. **公共思想输出**：将学习、研究、技术与反思沉淀为可被他人阅读的文本。
3. **未来可变现的个人品牌雏形**：为可能的副业、产品、出版或项目合作积累长期资产。

短期不追求流量与频率，长期追求：

> **在漫长时间中，不失真。**

---

## 二、站点结构与栏目设计

采用 Jekyll `_posts` 作为统一发布入口，所有正式文章必须进入时间轴系统。

### 栏目（Categories）

* `cn`：人生、自我、情感、孤独、时间、博士阶段的内在书写
* `en`：English learning notes, research logs, technical fragments
* `oracle`：星盘、周易、命运隐喻与诗性实验（非迷信，作为人生隐喻系统）
* `logs`：生活日志、运动、日常碎片

---

## 三、唯一正确的写作发布流程（SOP 核心）

### 1️⃣ 文件唯一存放位置

所有正式文章 **只允许放在：**

```
_posts/
```

禁止在以下位置发布正式文章：

* 根目录
* `cn/`、`en/` 等普通文件夹

---

### 2️⃣ 统一命名规则

```
YYYY-MM-DD-english-slug.md
```

示例：

```
_posts/2025-11-28-startingpoint.md
```

---

### 3️⃣ 统一文件头模板（必须项）

```md
---
layout: post
title: 中文标题
categories: cn
---
```

* `layout`：固定为 `post`
* `title`：显示在网页上的中文标题
* `categories`：对应栏目（cn / en / oracle / logs）

---

### 4️⃣ 正文书写规范

* 允许混合：中文 + 英文 + 数学公式
* LaTeX 公式写法：

```md
行内公式：$E = mc^2$

块级公式：

$$
\int_0^1 x^2 dx = \frac{1}{3}
$$
```

* 不追求“正确”，优先保证“真实”与“可追溯”

---

### 5️⃣ 发布与访问规则

发布后访问路径自动为：

```
https://wanchang97.github.io/<category>/YYYY/MM/DD/slug.html
```

示例：

```
https://wanchang97.github.io/cn/2025/11/28/startingpoint.html
```

---

## 四、内容层级原则（防止焦虑式输出）

所有内容默认分为三层：

1. **原始层**：碎片、情绪、困惑、不成熟想法（logs / cn）
2. **整理层**：结构化反思、可复读的经验总结（cn / en）
3. **实验层**：星盘、易经、命运叙事、人生隐喻（oracle）

任何一篇文章不要求立刻达到“整理层”。

允许：

> 先混乱 → 后澄明 → 再回看。

---

## 五、发布频率原则

* 不设周更、日更目标
* 只设一个长期原则：

> **在五年尺度上持续存在。**

哪怕一年只写 3 篇，只要是真实生长，也视为成功。
